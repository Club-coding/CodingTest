### pull&push
**항상 local에서 pull 먼저**
1. 개인 브랜치 이름(영문이니셜로 명명)
2. push local -> remote 자신의 브랜치
3. pull request로 풀이, 자신의 코드, 이슈 작성
4. 댓글로 의견 공유(코드 작성, 효율성 등)
5. 매번 4명의 승인

### 파일명 규칙
이름/날짜(yy/mm/dd)/문제 이름(영문) 또는 번호
예시) 김광채/210708/phoneNumberList.java
